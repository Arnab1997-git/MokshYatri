# Moksh Yatri Database Schema

## Overview

The platform uses Supabase as the primary database.

This document describes the major tables and their purposes.

---

# profiles

Purpose:

Stores user information and role-based access.

Key Fields:

- id
- full_name
- username
- role
- bio
- home_city
- travel_style
- travel_personality
- avatar_url

Roles:

- CUSTOMER
- ADMIN

---

# leads

Purpose:

Stores travel inquiries submitted by customers.

Key Fields:

- id
- customer_name
- phone
- destination
- budget
- travel_month
- travelers
- status
- source
- user_id

Statuses:

- NEW
- CONTACTED
- QUALIFIED
- CONVERTED
- CLOSED

---

# quotations

Purpose:

Stores quotations generated by travel consultants.

Key Fields:

- id
- lead_id
- quotation_number
- total_price
- status

---

# bookings

Purpose:

Stores confirmed customer bookings.

Key Fields:

- id
- quotation_id
- customer_id
- booking_status

---

# payments

Purpose:

Tracks customer payments.

Key Fields:

- id
- booking_id
- amount
- payment_status

---

# hidden_gems

Purpose:

Stores curated hidden travel locations.

Key Fields:

- id
- title
- location
- description
- category

---

# community_posts

Purpose:

Stores community travel stories.

Key Fields:

- id
- title
- content
- user_id
- created_at

---

# notifications

Purpose:

Stores customer notifications.

Key Fields:

- id
- user_id
- title
- message
- is_read

---

# hotel_allocations

Purpose:

Tracks hotel assignments for bookings.

---

# vehicle_allocations

Purpose:

Tracks vehicle assignments for bookings.

---

# reviews

Purpose:

Stores customer reviews and ratings.
